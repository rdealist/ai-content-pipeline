## 2. 内容SOP

### 2.1 选题流程

```
① 关键词调研（每周一次）
   ↓ 搜索：AI提效 / AI编程 / Claude / 程序员副业 / 效率工具
   ↓ 记录：点赞>200的笔记标题、封面风格、评论区高频问题
   
② 选题池维护（specs/xiaohongshu/ 目录）
   ↓ 每周从调研中提取3-5个选题
   ↓ 按「搜索量 × 竞争度低」排序
   
③ 内容生产（每条笔记30-60分钟）
   ↓ 写Spec → AI生成初稿 → 人工润色（去技术化）→ 配图
   ↓ 使用 prompts/xiaohongshu-note.md 模板
   
④ 发布 → 数据复盘
```

### 2.2 内容类型矩阵

| 类型 | 占比 | 示例标题 | 目标 |
|------|------|---------|------|
| **实测分享** | 40% | "Claude Code实测：3天搞定一周的活" | 引流+涨粉 |
| **工具安利** | 25% | "程序员必装的5个AI提效插件" | 搜索流量 |
| **经验观点** | 20% | "年薪30万程序员，下班做数字资产" | 共鸣+收藏 |

### 2.4 封面制作规范

- **尺寸**：1242×1660px（3:4竖版）
- **风格**：深色代码编辑器背景 + 大字白色/绿色标题
- **工具**：Canva / Figma 模板
- **元素**：
  - 左上角：个人IP标识（小logo或昵称）
  - 中间：核心标题（不超过12个字）
  - 下方：副标题或数据亮点
- **禁忌**：不要用纯文字封面，不要用模糊截图

## 4. 引流SOP（低调安全）

### 阶段一（0-500粉）：不主动引流
- 专注内容质量，积累平台权重
- 个人简介只留邮箱
- GitHub链接放在笔记正文中（小红书对GitHub链接审核松）

### 阶段二（500-1000粉）：软引流
- 发福利帖："私信【Skill】获取Claude Code技巧合集"
- 私信中引导到GitHub → GitHub README留邮箱
- 路径：小红书 → GitHub → 邮箱 → 微信

### 阶段三（1000+粉）：群聊引流
- 开通小红书群聊
- 群内发独家内容预告
- 群内自然过渡到微信群（"微信群有更多技术讨论"）
