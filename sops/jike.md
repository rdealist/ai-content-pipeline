## 2. 内容SOP

### 2.1 内容类型

| 类型 | 频率 | 字数 | 示例 |
|------|------|------|------|
| **日常观察** | 每天1条 | 100-300字 | "今天非技术同事接手了一个Node服务..." |
| **技术洞察** | 每周2条 | 300-500字 | "Spec驱动开发：为什么我不再从零写代码" |
| **项目更新** | 每周1条 | 200-400字 | "ai-content-pipeline v0.2发布..." |
| **转发评论** | 随时 | 50-150字 | 对热门技术话题发表观点 |

### 2.2 内容风格

即刻用户偏好：**真实、有观点、不装**

**DO ✅**
- 用口语化表达："今天发现一个有意思的事..."
- 带个人观点："我觉得大部分人用AI的方式都错了"
- 承认不确定："不太确定这个方向对不对，但先做了再说"
- 分享过程而非结果："昨晚花了3小时调Prompt, 终于..."

**DON'T ❌**
- 不要写公众号体（"众所周知"、"干货分享"）
- 不要硬广（"快来关注我的XX"）
- 不要纯转发不加观点
- 不要写太长（超过500字用长文功能）

### 2.3 动态模板

#### 日常观察模板
```
{场景描述：1-2句话}

{你的思考/观点：2-3句话}

{开放性问题或结论}
```

示例：
```
今天帮产品经理写了个数据查询脚本，Claude Code 10分钟搞定。

```

## 3. 发布SOP

### 3.1 发布时间
| 时段 | 适合内容 |
|------|---------|
| **8:00-9:00** | 通勤时的短观察 |
| **12:00-13:00** | 午休技术思考 |
| **21:00-22:00** | 总结类/项目更新 |

### 3.2 互动策略
- 每天浏览关注圈子的热门动态，回复3-5条
- 用"有价值的补充"而非"说得好"（即刻用户讨厌无意义互动）
- 对同行的项目真诚提问/建议，建立连接
- 被赞时主动回访对方主页，找到可互动的内容

### 3.3 每日流程
```
早上（5分钟）：
  □ 浏览关注动态，点赞/评论2-3条有价值内容
  □ 如果有灵感，发一条短观察

中午（10分钟）：
  □ 浏览圈子热门，回复有深度的讨论
  □ 发一条当日技术观察（如果有的话）

晚上（15分钟）：
  □ 回复当日评论/私信
  □ 如果有项目更新，发布更新动态
  □ 浏览"推荐"页面，拓展新连接
```

## 4. 引流策略

即刻的引流方式和小红书不同 — **不要主动引流，让人来找你**：

1. 个人主页放GitHub链接（即刻支持）
2. 项目更新动态自然带GitHub链接
3. 有人问"怎么做到的" → 回复"我写了个开源工具：{链接}"
4. 积累到一定关注后，发起"圈子"或"主题讨论"
