## 分发格式转换规则

### 长文 → 小红书笔记（3-5条）

**转换方法**：
1. 提取文章的3-5个核心要点
2. 每个要点单独成一条笔记
3. 用 `prompts/xiaohongshu-note.md` Prompt改写每条
4. 制作对应封面

**示例**：
```
原文：《Spec驱动开发实践：从需求到上线的AI辅助工作流》（3000字）

拆解为：
笔记1：「什么是Spec驱动开发？程序员效率翻倍的秘密」
笔记2：「AI写代码靠谱吗？我用了3个月的真实感受」
笔记3：「程序员必学的AI工作流 | 附完整模板」
```

### 长文 → 即刻动态（1-2条）

**转换方法**：
1. 提炼最有观点的那个段落
2. 用 `prompts/jike-post.md` 改写为口语化短动态
3. 文末自然附上原文链接或GitHub链接

### 长文 → 知乎回答

**转换方法**：
1. 找到知乎上相关问题
2. 将文章重点段落重组为回答结构（先结论，后展开）
3. 使用 `ai-publish format --to zhihu` 格式化

### 长文 → 小报童付费版

**转换方法**：
1. 在免费版基础上增加50%深度内容
2. 附加：完整代码/配置模板/踩坑细节
3. 仅在小报童发布完整版

## 内容资产沉淀检查清单

每篇内容产出后，检查是否完成以下资产化：

```
□ 原始Markdown存入 output/ 目录
□ Spec存入 specs/ 目录（可复用选题结构）
□ 如果调试了新Prompt，更新 prompts/ 目录
□ 各平台版本存入 output/{platform}/ 目录
□ 数据记录到 analytics/ 目录
□ 如果产出了新工具/脚本，提交到main分支
□ README更新（如有新功能）
```
